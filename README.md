O **Report App** é uma solução mobile robusta desenvolvida para centralizar a comunicação de falhas técnicas entre microempreendedores e a **Pensar Consultoria**. O projeto elimina o ruído de canais informais, garantindo que cada incidente seja documentado com evidências fotográficas e descrições técnicas precisas.

## Objetivo do Projeto

Desenvolvido como parte das Atividades de Extensão do curso de ADS (Estácio), este aplicativo visa melhorar a rastreabilidade de bugs e solicitações de suporte, permitindo que o consultor técnico tenha um histórico claro para a tomada de decisão e resolução de problemas.

## Tecnologias e Ferramentas

- **Linguagem:** Java (Android SDK)
- **Ambiente:** Android Studio (Emulação Pixel 10 Pro / API 37)
- **Versionamento:** Git (Padrão de documentação VF Corp)
- **Metodologia:** Scrum (Foco em entregas iterativas e qualidade)

## Funcionalidades Principais

| Funcionalidade | Impacto no Negócio |
| :--- | :--- |
| **Captura de Foto** | Geração de evidência visual imediata do erro. |
| **Descrição Detalhada** | Contextualização técnica para agilizar o diagnóstico. |
| **Interface Intuitiva** | Facilidade de uso para usuários não técnicos (clientes finais). |

## Qualidade e Testes (QA)

Como um projeto liderado por um **QA Analyst**, a aplicação passou por:

- **Sanity Tests:** Validação das funções críticas de câmera e envio de formulário.
- **UAT (User Acceptance Testing):** Homologação direta com o stakeholder (Pensar Consultoria).
- **Análise de Usabilidade:** Pesquisa quantitativa via formulários para medir a eficácia da ferramenta frente aos métodos tradicionais.
